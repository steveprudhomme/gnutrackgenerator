# Documentation

Ce dossier contient la documentation du projet, séparée du code source.

## Structure

- `architecture/` : décisions d’architecture, schémas, compromis techniques.
- `specifications/` : exigences fonctionnelles, formats, contrats et comportements attendus.
- `user/` : guides d’installation, d’utilisation et de dépannage.
- `legacy/` : documents historiques conservés pour référence.

La documentation doit évoluer avec le code. Toute modification fonctionnelle notable devrait inclure une mise à jour documentaire ou expliquer pourquoi elle n’est pas nécessaire.
