# Code de conduite

Ce code de conduite est inspiré du Contributor Covenant 2.1 et adapté au contexte du projet.

## Notre engagement

Nous nous engageons à faire de ce projet un espace accueillant, respectueux et constructif, peu importe l’expérience, l’identité, le niveau technique, la langue, l’origine, la situation personnelle ou le mode de contribution des personnes participantes.

La participation au projet est un privilège communautaire qui implique une responsabilité : contribuer à un environnement où les désaccords peuvent être exprimés sans intimidation, mépris ou harcèlement.

## Comportements attendus

Sont attendus :

- utiliser un langage accueillant et respectueux ;
- accepter la critique constructive ;
- se concentrer sur ce qui est meilleur pour le projet et ses utilisateurs ;
- reconnaître les erreurs et les corriger ;
- respecter le temps et les limites des autres ;
- aider les nouvelles personnes à s’orienter ;
- distinguer les désaccords techniques des attaques personnelles.

## Comportements inacceptables

Sont inacceptables :

- insultes, humiliation, intimidation ou menaces ;
- harcèlement public ou privé ;
- propos discriminatoires ou dégradants ;
- attention sexuelle non sollicitée ;
- publication d’informations privées sans consentement ;
- moquerie persistante d’un niveau d’expérience, d’une langue ou d’un handicap ;
- sabotage de discussions, tickets, revues ou outils communautaires ;
- pression pour obtenir du support gratuit ou immédiat ;
- représailles contre une personne ayant signalé un problème.

## Champ d’application

Ce code s’applique :

- au dépôt ;
- aux tickets et demandes de tirage ;
- aux discussions publiques ;
- aux salons de clavardage ;
- aux listes de diffusion ;
- aux événements liés au projet ;
- aux communications privées lorsqu’elles concernent directement le projet ou ses participants.

## Signalement

Les comportements inacceptables peuvent être signalés à : `[conduct@example.org]`

Un signalement devrait inclure, si possible :

- la description de l’incident ;
- les personnes impliquées ;
- les liens, captures ou journaux pertinents ;
- la date et le canal ;
- le niveau d’urgence ;
- toute mesure souhaitée pour la sécurité immédiate.

Les signalements seront traités avec discrétion. Les informations seront partagées seulement avec les personnes nécessaires à l’évaluation et à la résolution.

## Processus d’évaluation

1. Accusé de réception lorsque possible.
2. Évaluation initiale de la sécurité des personnes concernées.
3. Collecte minimale d’informations pertinentes.
4. Analyse par les responsables désignés du code de conduite.
5. Décision sur les mesures appropriées.
6. Communication de la décision aux personnes concernées, dans la mesure compatible avec la confidentialité.
7. Suivi pour prévenir les représailles.

Les personnes responsables d’un signalement doivent se retirer de l’évaluation si elles sont directement impliquées ou en conflit d’intérêts.

## Conséquences possibles

Selon la gravité, les conséquences peuvent inclure :

- rappel privé ou public des règles ;
- demande de modification de comportement ;
- retrait ou modification d’un commentaire ;
- fermeture d’un fil de discussion ;
- suspension temporaire de participation ;
- retrait de droits d’écriture ;
- exclusion permanente des espaces du projet ;
- signalement aux plateformes concernées si nécessaire.

Les mesures visent à protéger la communauté, non à humilier publiquement.

## Appel

Une personne visée par une mesure peut demander une révision à `[appeal@example.org]`, sauf en cas de risque immédiat pour la sécurité d’autrui.

L’appel doit expliquer les faits contestés, les éléments nouveaux ou les raisons pour lesquelles la mesure serait disproportionnée.

## Engagement contre les représailles

Les représailles contre une personne qui signale de bonne foi un incident ou participe à une enquête sont interdites et peuvent entraîner des mesures disciplinaires supplémentaires.
