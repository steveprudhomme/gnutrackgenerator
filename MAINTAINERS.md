## Responsabilités attendues

Les mainteneurs et mainteneuses doivent :

- appliquer le code de conduite ;
- protéger la qualité et la cohérence du projet ;
- expliquer les refus importants ;
- éviter les décisions opaques ;
- vérifier le respect de la licence GPLv3 et du DCO ;
- encourager les contributions nouvelles et raisonnablement cadrées ;
- protéger leur propre capacité bénévole afin d’éviter l’épuisement.

## Niveaux de responsabilité

Le projet peut accorder des droits progressivement :

1. **Tri** : catégoriser les tickets, demander des précisions, repérer les doublons.
2. **Revue** : relire les demandes de tirage et recommander des changements.
3. **Fusion limitée** : intégrer certains changements simples, par exemple documentation ou tests.
4. **Maintenance complète** : fusionner du code, publier des versions, participer aux décisions structurantes.

## Devenir mainteneur ou mainteneuse

Une personne peut être proposée lorsqu’elle a démontré :

- une contribution régulière ;
- une communication constructive ;
- une bonne compréhension des objectifs du projet ;
- un respect constant de `CONTRIBUTING.md`, `GOVERNANCE.md` et `CODE_OF_CONDUCT.md` ;
- une attention à la qualité, aux tests, à la documentation et à la sécurité juridique.

La nomination suit le processus décrit dans `GOVERNANCE.md`.
